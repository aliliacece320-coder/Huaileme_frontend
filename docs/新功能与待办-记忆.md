# 新开发功能位置与待办（供新对话使用）

## 一、新开发功能所在位置

### 1. 设置与登录相关（已存在文件）

| 功能 | 前端文件路径 | 路由 | 说明 |
|------|--------------|------|------|
| 设置页 | `src/views/SettingsPage.vue` | `/settings` | 底部五栏之一，入口：账号与家庭、通知设置、隐私与安全、关于我们 |
| 登录页（独立） | `src/views/LoginPage.vue` | `/login` | 独立全屏页，支持 `?redirect=` 回跳 |
| 账号与家庭 | `src/views/UserFamilyCenter.vue` | `/family` | 从设置页「账号与家庭」进入；未登录显示「去登录」→ `/login?redirect=/family` |
| 通知设置 | `src/views/NotificationSettingsPage.vue` | **未接路由** | 提醒、推送偏好（演示开关）；需在 `router` 增加路由并在设置页改为跳转 |

| 数据存储与同步 | `src/views/PrivacyDataPage.vue` | **未接路由** | 从隐私与安全页进入的子页；需接路由 |
| 关于我们 | `src/views/AboutUsPage.vue` | **未接路由** | 版本与介绍；需接路由 |

- **路由定义**：`src/router/index.js`。当前仅有 `/settings`、`/login`、`/family`，没有 `/settings/notifications`、`/settings/privacy`、`/settings/about`、`/settings/privacy/data`。

## 二、新对话需要处理的三个需求（用户已明确）

1. **新功能的后端与数据库文档**
   - 若新开发的这些部分需要**新的后端接口**或**新的数据库表定义**，请一并写到**单独的 md 文档**中。
   - 书写格式与现有 `API/` 下那几个 API 文档一致。

2. **零浪费页：去掉「钱包/钱」相关，只保留勋章、健康分、新鲜分**
   - 「钱包心疼指数」及对钱的统计**不需要**，可直接移除。
   - 当前只需要：**荣誉勋章**、**健康分**、**新鲜分**；前端只需改对应区块的**标题/展示**即可。

3. **注册要独立成单独页面并做好跳转**
   - 「注册新账号」应是**单独界面**，需要**跳转**进入，而不是和登录挤在同一屏。
   - 建议：登录页保留「登录」+ 一个「注册新账号」链接，点击后跳转到独立注册页（例如 `/register`），注册成功后再跳回登录页或首页；或登录页/注册页互相有入口，跳转逻辑清晰即可。

你可以直接对新对话说：

「本项目是 Vue3 移动端「坏了么」冰箱管理应用。请帮我做三件事：

1. **新功能文档**：新开发的设置子页（通知设置、隐私与安全、关于我们）、登录页、账号与家庭等，如果将来要接新后端接口或新表，请把接口说明和表结构写到一个单独的 md 里，格式和项目里 `API/` 下现有 API 文档一致。

2. **零浪费统计页**：去掉「钱包心疼指数」和所有跟钱相关的统计，只保留「荣誉勋章」「健康分」「新鲜分」，改一下前端标题和展示即可。文件在 `src/views/ZeroWasteStatisticsReport.vue`。

3. **注册独立页**：把「注册新账号」做成单独页面（例如 `/register`），从登录页跳转过去，并处理好注册成功后的跳转（如回登录页或首页）。登录页在 `src/views/LoginPage.vue`，路由在 `src/router/index.js`。」
